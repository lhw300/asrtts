为了防止混淆，这里梳理一下刚才提到的两种并发机制：

Uvicorn Workers (--workers)：这是在系统层面开启多个完全独立的 Python 进程。如果你设为 4，相当于开了 4 个监听同一个端口的服务器。适合压榨多核 CPU，但每个 Worker 都会把你的 ASR 和 TTS 模型重新加载一遍进内存！极其吃显存/内存。

Thread Pool Executor (我们在上一条加的 max_workers=64)：这是在单进程内部开启的异步线程池。15 路并发都在这一个进程里，模型只加载一次（省显存），遇到 C++ 同步计算时丢给这 64 个线程去排队执行。

ONNX Threads (--threads)：这是 C++ 底层推理引擎计算单句语音时使用的 CPU 核心数。

要在这三者之间取得完美平衡，我们需要明确它们各自的成本和收益，才能推导出一个“黄金配比”：第一层：Uvicorn Workers (多进程并行)作用：真正打破 Python GIL 限制，
在系统层面复制多个 FastAPI 实例，并行处理 WebSocket 连接。成本（极高）：内存/显存翻倍。每个 Worker 进程都会在内存里重新加载一份 SenseVoice 和 VITS 模型。
如果你有 4 个 Worker，你的模型内存占用直接 × 4。调优建议：不要开太多。假设你是 16 核 CPU，建议开 workers = 2 或 4。
这样既能利用多核处理 Python 逻辑，又不会把内存撑爆。这 4 个进程会通过操作系统的负载均衡，把 15 路并发平摊，每个进程同时只处理约 3-4 路通话。

第二层：内部 ThreadPoolExecutor (异步任务线程池)作用：让 FastAPI 在等待 C++ 语音模型推理（decode_stream 或 generate）和音频重采样（resample）时，主事件循环不被卡死，
继续接收其他用户的网络包。成本（极低）：线程是轻量级的，共享所属 Worker 进程的内存。
调优建议：由于我们已经用 Worker 分摊了请求，单个进程内的并发压力变小了。
建议每个 Worker 的最大线程池设为 max_workers = 16 即可。

第三层：ONNX Threads (--threads 算子级并行)作用：在 C++ 底层计算单句语音时，使用多少个 CPU 核心来加速矩阵运算。成本（极易引发灾难）：
CPU 线程上下文切换（Context Switching）。调优建议：在你的高并发场景下，这个值必须压低！致命错误公式：假设 workers=4，单进程分配 4 路并发，
如果你把 --threads 设为 4。那么极端情况下，底层会有 $4 \times 4 \times 4 = 64$ 个 CPU 密集型运算线程在同时抢占资源。
如果是 16 核 CPU，这会导致极高的切换损耗，处理速度反而变慢。

正确做法：为了保证总体吞吐量最大化，请强制将 --threads 设为 1 或 2。宁可单句话处理慢十几毫秒，也不能让 CPU 陷入调度地狱。??

1. 语音识别 (ASR) 与处理
sherpa-onnx-offline: 离线语音识别工具。你给它一个音频文件，它会输出文字。

sherpa-onnx-online-websocket-server: 实时语音识别服务器。支持通过 WebSocket 协议接收流式音频并实时返回识别结果。

sherpa-onnx-keyword-spotter: 关键词检测（唤醒词识别）。用于监听特定的词汇（如“嘿，Siri”）。

sherpa-onnx-vad: 语音活动检测 (Voice Activity Detection)。用于判断音频中哪一段是人在说话，哪一段是静音。

2. 语音合成 (TTS)
sherpa-onnx-offline-tts: 离线文字转语音。输入一段文本，生成语音文件。

sherpa-onnx-offline-zeroshot-tts: “零样本”语音合成。通常用于声音克隆，只需少量目标声音样本即可合成类似音色的音频。

3. 音频增强与分析
sherpa-onnx-offline-denoiser: 离线降噪工具，用于去除音频中的背景杂音。

sherpa-onnx-offline-speaker-diarization: 扬声器日志（话者分离）。用于分析一段录音中有几个人在说话，并标注出谁在什么时候说了什么。

sherpa-onnx-offline-language-identification: 语种识别。自动判断一段语音是哪种语言（如中文、英文、法文等）。

4. 其他辅助工具
sherpa-onnx-offline-punctuation: 自动加标点符号工具。

sherpa-onnx-offline-audio-tagging: 音频打标签（例如识别背景声是“狗叫”还是“鸣笛”）。

sherpa-onnx-version: 查看当前软件的版本信息。
